What is an API?


API is an acronym for “Application Programming Interface,” and it is a set of well defined protocols that can be used by developers and programers to build applications and interact with data.

APIs were developed in order to make it easier for one application to communicate with another application, without either application knowing how the other one was built or implemented. Essentially, it facilitates the sharing of information in a standardized way. Application 1 sends a request for information structured in a particular way, then application 2 will respond in another particular way. Developers can use these predictable, standardized responses to build new tools on top of existing applications.

Typically, the structure most APIs use is called JSON, which stands for JavaScript Object Notation. Despite having the word “JavaScript” in its name, JSON is not specific to any particular coding language or application. JSON formatted data is presented as a simple series of key-value pairs, and it’s extremely flexible and extensible.

RedHat has an excellent guide to APIs if you would like to learn more.


Glass Plates Inventory API


The API of the Glass Plates Inventory site allows users to list all archives currently in the database, view details about a specific archive, browse the plate collections of each archive, search for plates, and view details of a specific plate. The API can be used to download data about multiple plates all at once in a programmatic fashion, enabling a user to collect, manipulate, and explore the data in whatever method is best suited for their use.

Sample Queries


All plates with the Horsehead Nebula:
/api/plates?archive=all&limit=50&skip=0&obj=horsehead+nebula

All plates at the WFPDB that cover RA 13h 33m:
/api/plates?archive=wfpdb&limit=50&skip=0&ra=13:33

All plates at DASCH with a note indicating they are broken:
/api/plates?archive=dasch&limit=50&skip=0&text=broken
